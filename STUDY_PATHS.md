# Study Paths

This file helps you choose the right training path inside PFPython.

## Quick Start

If you want the shortest useful path:

1. Finish all files in `foundations/`
2. Finish `core_python/`
3. Finish `modules/`
4. Finish `engineering/01` through `engineering/05`
5. Build one project from `projects/`

## Weekly Pattern

Use this weekly rhythm for any path:

1. Learn: run and read 1 to 3 files.
2. Modify: change values, logic, or structure.
3. Rebuild: rewrite one example without looking.
4. Review: rerun an older lesson.
5. Apply: connect the idea to a project file.

## How To Measure Progress

Use simple checkpoints:

- Can I explain this file in plain English?
- Can I rewrite the main idea from memory?
- Can I change the code without breaking it?
- Can I use the same idea in a different file?
- Can I write a tiny version on my own?

## Good Habits

- Keep a daily practice note.
- Predict outputs before running code.
- Rename variables and refactor small pieces.
- Add one extra example after every lesson.
- Revisit hard files after one week.
