# 90 Day Plan

This plan is for deep and steady training.

Goal:

- build strong Python fluency
- gain interview practice
- gain engineering habits
- feel comfortable reading and extending real code

## Month 1: Syntax And Core Language

Focus on:

- all `foundations/`
- all `core_python/`
- all `modules/`
- all `pfpython/`

Targets:

- explain every file clearly
- rebuild 10 files from memory
- create 5 personal variations

## Month 2: Libraries, Testing, And Structure

Focus on:

- all `standard_library/`
- all `engineering/`
- all `packaging/`
- all `tests/`

Targets:

- understand file I/O and data formats
- understand config, logging, retry, mocking, and tests
- understand how package structure supports reuse

## Month 3: Advanced Practice, Interview Work, And Projects

Focus on:

- all `advanced/`
- all `interview/`
- all `projects/`

Targets:

- understand async and queue-based thinking
- practice interview-style explanations
- extend at least 2 projects with your own features

## Suggested Weekly Split

Monday:
- new lessons

Tuesday:
- more lessons and output prediction

Wednesday:
- refactor day

Thursday:
- tests and debugging

Friday:
- project work

Saturday:
- interview practice

Sunday:
- review and rewrite day

## Milestones

By Day 30:

- finish `foundations/`
- finish `core_python/`
- finish `modules/`

By Day 60:

- finish `standard_library/`
- finish `engineering/`
- finish `tests/`
- finish `packaging/`

By Day 90:

- finish `advanced/`
- finish `interview/`
- finish `projects/`
- create 1 original multi-file project of your own

## Final Challenge

At the end of 90 days:

1. Create a new project folder.
2. Add config, logging, persistence, and tests.
3. Use at least one queue, one retry pattern, and one custom helper module.
4. Document how to run it in English.
